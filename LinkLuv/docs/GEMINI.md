# Gemini CLI - 專案設定與客製化指南 (LinkLuv)

本指南記錄了使用 Gemini CLI 在 LinkLuv 專案中進行的關鍵設定與功能開發。

### 2.4 輪播介面優化
*   **箭頭導航**: 移除瀏覽器預設捲動條，改用兩側自定義箭頭導航按鈕，增強互動體驗。
*   **垂直置中方案 (CSS 優化)**: 捨棄了傳統的 `top-1/2 -translate-y-1/2` 絕對定位方案，改為採用父容器 `flex items-center` 配合 `absolute` 定位的機制，確保左右箭頭在不同內容高度下均能嚴格垂直居中，大幅降低佈局偏移風險。
### 2.5 留言系統階梯式 UI 與圖片擴充
*   **階梯式渲染**: 使用遞迴 Blade 渲染邏輯，依據 `depth` 欄位動態計算縮排 (`margin-left`)，實現視覺化的階梯式回覆鏈，並限制最大深度為 30 層。
*   **卡片化設計**: 將留言渲染結構全面改為獨立卡片模式，整合用戶資訊、留言內容、圖片展示與內嵌回覆表單，提升資訊結構化程度。
*   **圖片功能**: 擴充 `messages` 表格新增 `image_path` 欄位，支援留言與回覆時的 Multipart 檔案上傳。

## 2. 留言系統強化功能

為了提升社群互動體驗，本次開發針對留言牆進行了多項功能升級：

### 2.1 圖片上傳功能
*   **欄位擴充**: 在 `messages` 資料表中新增了 `image_path` 欄位。
*   **檔案處理**: 在 `MessageController` 的 `store` 方法中整合了 `Illuminate\Http\Request` 的檔案上傳邏輯，將圖片儲存於 `storage/app/public/messages`。
*   **儲存連結**: 透過 `php artisan storage:link` 建立軟連結，並在 Blade 視圖中使用 `asset()` 輔助函數渲染圖片。

### 2.2 階層式回覆機制
*   **階層與數量限制**: 嚴格執行回覆層級與該層回覆數量上限皆為 30 層，防止過度嵌套。

### 2.3 介面佈局優化
*   **左對齊配置**: 調整了 `feed.blade.php` 的 CSS 佈局，將所有動態內容區塊統一為左對齊，符合閱讀慣性。
*   **響應式網格**: 留言牆採用 CSS Grid 佈局 (`grid-cols-1 md:grid-cols-2 lg:grid-cols-3`)，實現每列三個卡片的響應式呈現。
*   **精選動態輪播**: 重構了精選動態卡片輪播系統，移除了預設捲動條，改用兩側自定義箭頭導航按鈕，並整合了 JavaScript 自動捲動機制（每 5 秒切換）。
### 2.4 輪播介面優化
*   **箭頭導航**: 移除瀏覽器預設捲動條，改用兩側自定義箭頭導航按鈕，增強互動體驗。
*   **垂直置中方案 (CSS 優化)**: 捨棄了傳統的 `top-1/2 -translate-y-1/2` 絕對定位方案，改為採用父容器 `flex items-center` 配合 `absolute` 定位的機制，確保左右箭頭在不同內容高度下均能嚴格垂直居中，大幅降低佈局偏移風險。
