# 人物氣泡框與連線設計指南

本文件記錄了 LinkLuv 留言系統的「氣泡框與連線 (Connector)」完整架構，包含最新的 JavaScript 渲染邏輯與 UI 設計標準。

## 1. 核心渲染架構 (JavaScript)
我們採用「分組 (Thread)」架構，捨棄了單純 CSS 縮排，改用遞迴渲染，確保連線能準確從根留言延伸至回覆留言。

### 1.1 主渲染函式
此函式將後端返回的扁平數據轉為 Thread 分組，並呼叫渲染組件。

```javascript
function renderMessages(messages) {
    const list = document.getElementById('messages-list');
    list.innerHTML = '';
    
    const threads = [];
    const threadMap = new Map();

    messages.forEach(msg => {
        if (msg.depth === 0) {
            const thread = { root: msg, replies: [] };
            threads.push(thread);
            threadMap.set(msg.id, thread);
        } else {
            const rootThread = threadMap.get(msg.thread_id);
            if (rootThread) rootThread.replies.push(msg);
        }
    });

    threads.forEach(thread => {
        list.insertAdjacentHTML('beforeend', buildThread(thread.root, thread.replies));
    });
}
```

### 1.2 遞迴組件渲染 (繪製連線)
透過 `buildThread` 與 `buildReplyHTML` 繪製連線：
* **縱線 (vLine)**：在根留言區塊左側繪製，高度隨回覆數量自動調整。
* **橫線 (hLineWidth)**：在回覆區塊繪製，長度根據 `depth` 動態運算：`16 + (depth - 1) * 28`。
* **頭像縮放**：`avatarSize = Math.max(24, 30 - (depth - 1) * 3)`，確保留言內容有足夠寬度。

---

## 2. 進階架構思考 (優化點)
* **渲染機制**：放棄了單純的 CSS `margin-left`，改用「遞迴渲染函式 (buildThread)」。
    * **優點**：確保每一則回覆都能精準連接到父留言，解決了深層留言排版崩潰的問題。
* **動態計算**：透過後端提供的 `depth` 與 `path` 欄位，在前端動態計算線條寬度 (`hLineWidth`)，實現了彈性的響應式排版。
* **可維護性**：將渲染邏輯封裝為組件函式，避免了傳統 Blade 模板中混寫大量邏輯。

---

## 3. 實作檢查清單 (feed.blade.php)

在實作此視圖概念時，請確保：

### 3.1 渲染邏輯一致性
* **清理舊代碼**：已移除舊版 `renderMessages` 中基於 `marginClass` 的循環渲染，改由 `buildThread` 接管。
* **輔助函式**：確認 `buildThread`, `buildRootHTML`, `buildReplyHTML` 這三個函式完整定義於 `<script>` 區塊中。
* **資料依賴**：後端返回的物件必須包含 `depth` 與 `thread_id` 欄位。

### 3.2 HTML 結構與樣式
* **Flex 結構**：回覆留言的 `div` 必須包含 `style="display:flex; align-items:flex-start;"`，並透過 `margin-left:-16px` 對齊連線。
* **圖片路徑**：確保 `imageHtml` 的渲染路徑 `/storage/${msg.image_path}` 對應正確。

### 3.3 事件繫結
* 確保 `toggleReply`, `deleteMessage`, `editMessage`, `toggleLike` 等功能按鈕都能正確綁定 `msg.id`。
