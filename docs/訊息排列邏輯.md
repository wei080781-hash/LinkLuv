# 留言系統：Facebook 風格連線 UI 實作指南

本文件指導如何將現有的留言系統渲染邏輯，重構成具有「氣泡框與連線 (Connector)」的 Facebook 風格 UI。

## 1. 架構核心變更
本實作捨棄了單純的 CSS 縮排 (`ml-x`)，改採 **「JavaScript 遞迴渲染 (Building Threads)」** 架構。

*   **分組邏輯**：後端將留言按 `thread_id` 聚合，前端再依據 `depth=0` 進行分串處理。
*   **視覺連線**：使用絕對定位繪製垂直與水平線條，確保視覺上的階層感穩定。
*   **頭像動態化**：隨層級增加縮小頭像尺寸，節省水平空間。

---

## 2. 前端實作步驟

### 步驟 1: JavaScript 渲染邏輯
在 `resources/views/feed.blade.php` 的 `<script>` 區塊中，將原本的 `renderMessages` 取代為以下結構：

#### A. 主渲染函式
此函式將留言依根留言進行分組，並交由 `buildThread` 處理。

```javascript
function renderMessages(messages) {
    const list = document.getElementById('messages-list');
    list.innerHTML = '';
    
    // 依「根留言」分組
    const threads = [];
    let currentThread = null;
    messages.forEach(msg => {
        if (msg.depth === 0) {
            currentThread = { root: msg, replies: [] };
            threads.push(currentThread);
        } else if (currentThread) {
            currentThread.replies.push(msg);
        }
    });

    threads.forEach(thread => {
        list.insertAdjacentHTML('beforeend', buildThread(thread.root, thread.replies));
    });
}
```

#### B. 遞迴組件渲染
您需要將您提供的三個輔助函式 (`buildThread`, `buildRootHTML`, `buildReplyHTML`) 放入 `feed.blade.php` 的 `<script>` 中。

---

## 3. UI 關鍵調整細節

### 3.1 縱線 (Vertical Connector)
在 `buildThread` 中，透過 CSS `flex` 容器來繪製縱線：
*   **實現方式**：一個寬度 10px 的容器，內含一個圓點與一條縱線 (`vLine`)。
*   **邏輯**：當 `replies.length > 0` 時，渲染該縱線，長度撐滿整個串。

### 3.2 橫線 (Horizontal Connector)
在 `buildReplyHTML` 中處理回覆的層級對齊：
*   **計算邏輯**：`hLineWidth = 16 + (depth - 1) * 28;`
*   **視覺效果**：隨著 `depth` 增加，橫線長度自動拉長，使氣泡框完美對齊。

### 3.3 頭像動態縮放
為避免層級過深時內容被擠壓，請務必在 `buildReplyHTML` 中加入：
```javascript
const avatarSize = Math.max(24, 30 - (depth - 1) * 3);
```
這能確保 Depth 3 之後的留言仍保有足夠的閱讀寬度。

---

## 4. 維護注意事項
1.  **資料預處理**：確保 API 返回的數據中包含 `depth` 欄位（若無，需在前端先依 `parent_id` 預先計算）。
2.  **安全性**：由於使用 `insertAdjacentHTML`，請確保從後端獲取的 `msg.content` 已經過適當處理（Laravel Blade 的 `{{ }}` 或 JSON 處理已提供基本防護）。
3.  **效能**：此架構為純前端渲染，適合百筆以內的留言。若留言數龐大，建議在渲染前進行分頁處理。
