# LinkLuv 專案功能白皮書

## 一、專案概述

LinkLuv 是一個基於 Laravel 的社交互動平台原型，核心聚焦於留言系統、回覆層級、圖片上傳、按讚機制與使用者帳戶管理。

本白皮書旨在清晰說明專案架構、功能分類、資料模型、流程設計與未來擴充方向，方便開發團隊檢視現有功能並進行優化。

---

## 二、系統分類

### 1. 身分驗證與帳戶管理
- 使用 Laravel 原生帳號註冊、登入、登出。 
- 支援 Email 驗證與密碼重設。
- 會員可編輯個人資訊與刪除帳戶。

### 2. 留言系統
- 使用者可發表留言。
- 留言支援階層回覆（parent-child relationship）。
- 留言可附帶圖片上傳。
- 留言可編輯與刪除。

### 3. 回覆與階層結構
- `Message` 模型支援 `parent_id` 與 `replies()` 關聯。
- 支援「根留言」和「回覆留言」的階層呈現。
- 目前 `MessageController@index` 依 `thread_id` 或 `id`、`created_at` 排序，以維持對話線性順序。

### 4. 點讚機制
- 留言可被登入使用者點讚或取消點讚。
- 點讚使用 `likes` 關聯表管理。
- API 回傳當前點讚數、是否已點讚，並將 `like` 路由納入 `auth` 保護。

### 5. 前端視圖與互動
- 使用 Blade 模板渲染 `welcome`、`feed`、`profile` 等頁面。
- `feed` 頁面透過 AJAX/Fetch 連接 `/api/messages`，即時顯示留言串。
- 已移除同好圈與默契配對頁面，前端主打生活牆功能。
- 按鈕與回覆表單以 `flex` UI 佈局呈現。

### 6. 支援性文件與開發指南
- `docs/MESSAGE_FEATURE.md`、`docs/LIKE_FEATURE.md`、`docs/DELETE_MESSAGE.md`、`docs/CLOSURE_TABLE.md` 等，用於記錄功能開發與排查流程。

## 三、核心資料模型

### 1. users
- Laravel 基本會員資料表。
- 欄位：`name`, `email`, `password`, `email_verified_at`, `remember_token`。

### 2. messages
- 儲存留言內容與回覆結構。
- 重要欄位：
  - `user_id`：留言作者。
  - `content`：留言文字內容。
  - `parent_id`：父留言 ID，若為空則為根留言。
  - `thread_id`：回覆 thread 追蹤欄位，用於線性排序與分組。
  - `image_path`：圖片上傳後的儲存路徑。
  - `depth`：用於前端縮排或階層展示。
- `Message` 模型已啟用 `SoftDeletes`，支援軟刪除。

### 3. likes
- `likes` 表記錄「使用者對留言的點讚關聯」。
- 重要欄位：`user_id`, `message_id`。
- `Message` 與 `Like` 建立一對多關聯。

### 4. message_closure
- 作為可選擇的階層結構優化方案。
- 使用閉包表模型儲存祖先-後代關係，可高效查詢所有子孫節點。
- 適合深層回覆樹、複雜留言樹查詢。

## 四、流程設計

### 1. 註冊與登入
1. 使用者進入註冊頁面。
2. 提交姓名、Email、密碼。
3. 系統建立帳戶並登入，可能觸發 Email 驗證流程。
4. 已登入使用者可訪問 `feed`。

### 2. 發表留言
1. 使用者在 `feed` 頁面填寫留言內容與可選圖片。
2. 後端 `MessageController@store` 驗證：
   - `content` 是必填且長度不超過 1000。
   - `image` 可空，若上傳則驗證為圖片且不超過 2MB。
   - `parent_id` 若存在，必須是有效的 `messages.id`。
3. 若為回覆留言，計算 `thread_id` 並保存。
4. 儲存後回傳 JSON 成功響應。

### 3. 顯示留言串
1. 前端向 `/api/messages` 送出 GET 請求。
2. `MessageController@index` 取出留言、作者關聯，並排序。
3. JSON 包含留言內容、作者、點讚數、是否已點讚等資訊。
4. 前端渲染留言，依 `depth` 或 `thread_id` 顯示縮排與回覆階層。

### 4. 點讚與取消點讚
1. 使用者按下讚按鈕時，前端呼叫 `POST /messages/{message}/like`。
2. `MessageController@like` 依登入使用者判斷是否已存在 `Like`。
3. 若已點讚則刪除；否則新增。
4. 回傳 `liked` 狀態與最新 `likes_count`。

### 5. 編輯留言
1. 登入使用者僅能修改自己的留言。
2. `MessageController@update` 進行權限檢查與驗證。
3. 成功更新後回傳 JSON 成功值。

### 6. 刪除留言
1. `MessageController@destroy` 先驗證留言擁有者。
2. 刪除留言前，先刪除該留言的直接回覆。
3. 若採用閉包表，則可擴充為同時刪除所有子孫關係。
4. 回傳 JSON 成功結果。

## 五、系統架構與技術棧

### 1. 技術棧
- PHP + Laravel
- Blade 模板
- Eloquent ORM
- Laravel Auth (含 `auth.php` 路由)
- JavaScript Fetch API
- 文件儲存 (`storage/app/public`) 及 `storage:link`

### 2. 路由規劃
- `/`：歡迎頁面
- `/dashboard`：登入後重新導向至留言牆（`/feed`）
- `/feed`：留言牆
- `/profile`：使用者個人資料編輯
- `/profile/{id}`：使用者公開頁面
- `/messages`：留言建立、更新、刪除
- `/api/messages`：留言列表 API
- `/messages/{message}/like`：點讚API

### 3. 安全與權限
- 許多操作受 `auth` middleware 保護，僅限登入使用者操作。
- `MessageController` 的 `update`、`destroy`、`like` 都檢查目前登入者是否符合操作權限。
- 前端 CSRF Token 驗證為必要，以保護 AJAX 動作。

### 4. 前端互動設計
- 留言牆採用 AJAX 方式重新整理留言。
- 表單送出時使用 `FormData` 以支援圖片上傳。
- 操作按鈕應置於同一 `flex` 容器，保持 UX 一致。

## 七、建議優化方向

1. **全面補齊 `likes` 與 `is_liked` 的 API 資料回傳**，強化前端即時顯示。
2. **完整實作 `message_closure`**，若專案希望支援深度回覆與高效查詢，可將閉包表納入核心資料結構。
3. **統一前端按鈕排列**：將回覆、刪除、讚放入同一 `flex` 容器。
4. **優化路由層次**：將所有留言操作納入 `auth` 群組，避免權限遺漏。
5. **建立測試文件**：增加 `MessageController` 單元測試與整合測試，特別是留言建立、編輯、刪除、點讚流程。

## 八、功能代碼詳解與頁面說明

### 1. 留言發表功能（`resources/views/components/message-form.blade.php`）
- 頁面上使用 `message-form` 元件呈現留言輸入框與圖片上傳欄位。
- 表單 HTML：
```html
<form onsubmit="submitPost(event)" class="message-form" enctype="multipart/form-data">
    @csrf
    <textarea name="content" required placeholder="說點什麼..." class="w-full border-gray-200 rounded-lg focus:ring-pink-500 focus:border-pink-500 shadow-sm"></textarea>
    <input type="file" name="image" class="mt-3 text-sm text-gray-500">
    <button type="submit" class="bg-pink-600 text-white px-6 py-2 rounded-full">送出</button>
</form>
```
- `enctype="multipart/form-data"` 是關鍵，因為 `FormData` 需要上傳圖片檔案。
- 提交後由 `submitPost(event)` 透過 `fetch` 將資料送至 `POST /messages`。

### 2. 留言儲存後端邏輯（`app/Http/Controllers/MessageController.php`）
- `store(Request $request)` 負責驗證、處理圖片、計算回覆鏈、儲存留言：
```php
$validated = $request->validate([
    'content' => 'required|string|max:1000',
    'image' => 'nullable|image|max:2048',
    'parent_id' => 'nullable|exists:messages,id',
]);

$parentId = $request->input('parent_id');
$threadId = null;
if ($parentId) {
    $parent = Message::find($parentId);
    $threadId = $parent ? ($parent->thread_id ?? $parentId) : null;
}

$imagePath = $request->hasFile('image') ? $request->file('image')->store('messages', 'public') : null;

Message::create([
    'user_id' => auth()->id(),
    'content' => $request->content,
    'parent_id' => $parentId,
    'thread_id' => $threadId,
    'image_path' => $imagePath,
]);
```
- `parent_id` 代表回覆目標；若存在則會繼承父留言的 `thread_id`，使回覆與原始討論串綁在一起。
- `image_path` 儲存檔案位置，前端可用 `/storage/${msg.image_path}` 顯示圖片。

### 3. 留言列表與回覆呈現（`resources/views/feed.blade.php`）
- `feed.blade.php` 透過 `loadMessages()` 向 `/api/messages` 請求 JSON 資料。
- `renderMessages(messages)` 將留言列表渲染到 `<div id="messages-list">`：
  - 根留言 `parent_id === null` 無縮排；回覆留言加上 `ml-12`。
  - 若 `msg.image_path` 存在，會在留言內容下方插入 `<img>` 標籤。
- 每則留言帶操作按鈕：回覆、刪除、編輯、點讚。
- 回覆表單 `reply-form-{msg.id}` 初始隱藏，點擊「回覆」按鈕呼叫 `toggleReply(id)` 顯示。

### 4. 點讚機制（`MessageController@like` 與 `feed.blade.php`）
- `MessageController@like(Message $message)` 檢查登入使用者是否已對該留言存在 `Like`：
```php
$like = $message->likes()->where('user_id', auth()->id())->first();
if ($like) {
    $like->delete();
    $isLiked = false;
} else {
    $message->likes()->create(['user_id' => auth()->id()]);
    $isLiked = true;
}
return response()->json([
    'success' => true,
    'liked' => $isLiked,
    'likes_count' => $message->likes()->count(),
]);
```
- `feed.blade.php` 中的 `toggleLike(messageId)` 以 `POST /messages/{message}/like` 切換點讚狀態。
- 前端重載留言列表後，按讚數與樣式會同步更新。

### 5. 留言編輯（`editMessage` / `saveEdit`）
- 使用者按「編輯」時，`editMessage(messageId)` 會把留言內容替換成 `textarea`：
```js
messageDiv.innerHTML = `
    <textarea id="edit-textarea-${messageId}" class="w-full text-sm border rounded-lg p-2">${originalContent}</textarea>
    <button onclick="saveEdit(${messageId})">儲存</button>
`;
```
- `saveEdit(messageId)` 傳送 `PATCH /messages/{message}`：
```js
fetch(`/messages/${messageId}`, {
    method: 'PATCH',
    headers: {
        'Content-Type': 'application/json',
        'X-CSRF-TOKEN': ...
    },
    body: JSON.stringify({ content: newContent })
})
```
- 後端 `MessageController@update` 先檢查 `auth()->id() === $message->user_id`，再驗證 `content`，最後更新留言內容。

### 6. 留言刪除（`deleteMessage` 與 `MessageController@destroy`）
- `deleteMessage(messageId)` 呼叫：
```js
fetch(`/messages/${messageId}`, {
    method: 'DELETE',
    headers: {
        'X-CSRF-TOKEN': ...
    }
})
```
- `MessageController@destroy` 先檢查是否為留言擁有者，若通過則刪除該留言與其直接 `replies()`：
```php
if ($message->user_id !== auth()->id()) {
    return response()->json(['success' => false, 'message' => '無權刪除'], 403);
}
$message->replies()->delete();
$message->delete();
```
- 目前採用軟刪除 `SoftDeletes`，若資料表有 `deleted_at` 則可回復或做後續資安查核。

### 7. 登入後導向與留言牆
- 登入後使用者會導向 `feed` 留言牆。
- `/dashboard` 目前保留為舊的登入導向別名，並重新導向到 `/feed`。
- `feed.blade.php` 為主要互動頁面，透過 AJAX 取得並渲染留言列表。

### 8. 個人資料頁面與帳戶管理（`ProfileController`）
- `ProfileController@edit` 讓登入使用者載入編輯頁面：
```php
return view('profile.edit', ['user' => $request->user()]);
```
- `update` 方法會驗證輸入，若 Email 被更改則重設 `email_verified_at`。
- `destroy` 會驗證密碼、登出使用者、刪除帳戶並重整 Session。
- 對應 Blade 檔案：`resources/views/profile/edit.blade.php`、`profile/partials/*`。

### 9. 路由與權限
- 主要留言互動路由集中於 `routes/web.php` 的 `auth` 群組：
```php
Route::middleware('auth')->group(function () {
    Route::post('/messages', [MessageController::class, 'store'])->name('messages.store');
    Route::get('/api/messages', [MessageController::class, 'index']);
    Route::patch('/messages/{message}', [MessageController::class, 'update'])->name('messages.update');
    Route::delete('/messages/{message}', [MessageController::class, 'destroy'])->name('messages.destroy');
    Route::post('/messages/{message}/like', [MessageController::class, 'like'])->name('messages.like');
});
```
- `like` 路由已納入 `auth` 保護，確保僅登入使用者可以執行點讚。

## 九、結論

LinkLuv 已具備社交留言系統的核心骨幹，包含留言發表、回覆、編輯、刪除、點讚與帳戶管理。下一步應優先釐清留言樹深度與效能需求，並將所有互動操作轉為穩定的 API 流程，同時補上完整測試與文件。
